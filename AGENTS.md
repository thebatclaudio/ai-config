# AGENTS.md

Authoritative registry and operating manual for every AI agent, command, skill,
and MCP server defined in this repository. OpenCode loads this file (via the
`instructions` array in `opencode.json`) on every session, so keep it concise,
accurate, and current.

---

## 1. Project Context

- **Repository purpose:** Centralised, version-controlled OpenCode configuration
  (agents, commands, skills, MCP servers) bridged into the local OpenCode
  runtime via symlinks generated by `setup.py`.
- **Source of truth:** This repository. Never edit files inside
  `~/.config/opencode/` directly — edit them here and they propagate via the
  symlinks.
- **Configuration generator:** `setup.py` reads `.env`, renders
  `opencode.json.example`, and merges the result into the active
  `opencode.json` (existing values win on conflict).

---

## 2. Conventions

| Concern        | Rule                                                              |
| -------------- | ----------------------------------------------------------------- |
| File naming    | `kebab-case.md` for agents/commands; `snake_case.py` for skills. |
| One per file   | Exactly one agent/command per file.                               |
| Frontmatter    | Use OpenCode YAML frontmatter where applicable.                   |
| Documentation  | Every entry below MUST have a doc block in section 4.             |
| Lifecycle      | Add → document → test → commit. Remove the doc block when retiring an entry. |

## 4. Agent Doc Blocks

Use the template below for every agent. Duplicate it as needed.

### Template

```
### <agent-name>

- **File:** `agents/<agent-name>.md`
- **Purpose:** One sentence describing why this agent exists.
- **Capabilities:**
  - Bullet list of what the agent can do.
- **Constraints:**
  - Bullet list of what the agent must not do.
- **Tools:** `read`, `edit`, `bash`, ... (explicit allow-list)
- **Model:** `<provider>/<model-id>`
- **System Prompt Summary:** 2–3 sentences distilling the prompt.
- **Example Invocations:**
  - User: "..." → Agent: "..."
```

## 8. Operational Notes

- **Secrets:** Never hard-code API keys. Reference them via environment
  variables defined in `.env`.
- **Determinism:** Prefer pinned model IDs over moving tags.
- **Audit trail:** Significant prompt changes should be summarised in the
  commit message.
- **Recap warnings:** When `setup.py` reports merge collisions, reconcile them
  promptly — divergence between this repo and `~/.config/opencode/opencode.json`
  is a smell.
